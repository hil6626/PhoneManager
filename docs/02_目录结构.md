# 02 目录结构（标准化，合并自项目结构框架和依赖项清单.md）

**参考01总览**：定义 PhoneManager/ 项目目录，确保模块职责单一、可扩展。

## 标准目录树
```
PhoneManager/  # 新建项目根
├── main.py                    # 入口：GUI启动，import modules/ui
├── requirements.txt           # 依赖清单
├── README.md                  # 简要说明，链接docs/01总览
├── config/
│   └── config.json            # DB路径、日志级别（JSON，必填）
├── database/
│   └── data.db                # SQLite，主表+重复表+批次表
├── modules/                   # 业务逻辑（纯函数，无UI）
│   ├── recorder.py ✅         # 单/批量录入+去重完整 (add_single_number/add_batch pandas循环, data.db) [x implemented]
│   ├── query.py ✅             # 查询/生命周期 (lookup_single/batch, data.db只读) [x implemented]
│   ├── assigner.py ✅         # 分配/批量更新 (implemented)
│   └── stats.py               # 报表生成/导出
├── ui/
│   ├── main_window.py         # 主界面布局/信号槽
│   └── widgets/               # QTableWidget/日期选择等
├── resources/
│   ├── icons/                 # app.ico
│   └── templates/             # Excel报表模板.xlsx
└── logs/
    └── app.log                # 运行/异常日志
```

## 依赖清单 (requirements.txt 示例内容)
```
PyQt5==5.15.10
pandas==2.1.1
openpyxl==3.1.2
pyinstaller==5.14.0
```

## 注意事项
- **创建命令**：`mkdir -p PhoneManager/{config,database,modules,ui/widgets,resources/{icons,templates},logs}`
- **职责单一**：每个文件<200行，modules纯逻辑，ui纯界面。
- **扩展**：后期加 tests/ 单元测试。
- **冲突避免**：无 PhoneManager/ 则创建，现有 docs/ 独立。

## 检查命令
```bash
tree PhoneManager/ || find PhoneManager -type f | wc -l  # 应有10+文件位点
ls modules/ ui/  # 空(待开发), widgets/存在; find . -name '*.py' | grep -v __init__  # 验证main.py等
```

**防护规则**：
- 路径相对，确保exe打包 portable。
- Git忽略：data.db, logs/, *.pyc。
